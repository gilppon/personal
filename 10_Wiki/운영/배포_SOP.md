# 배포 SOP (표준 운영 절차)

> 서비스 배포 시 따르는 표준 절차

## 🚀 배포 유형별 SOP

### A. 정기 배포 (기능 추가/개선)
```
1. [개발] 기능 구현 완료 → 로컬 테스트
2. [리뷰] 코드 셀프 리뷰 (체크리스트 기반)
3. [빌드] npm run build → 에러 0건 확인
4. [배포] git push → CI/CD 자동 빌드
5. [검증] Preview URL에서 핵심 기능 테스트
6. [승인] 대표님 확인 (스크린샷/영상 첨부)
7. [출시] Production 배포
8. [감시] 5분간 에러 로그 모니터링
9. [기록] decisions.md에 배포 기록
```

### B. 핫픽스 배포 (긴급 버그)
```
1. [확인] 버그 재현 → 영향 범위 파악
2. [수정] 최소 범위 코드 수정
3. [테스트] 해당 기능 + 연관 기능 테스트
4. [배포] 즉시 Production 배포
5. [검증] 수정 사항 라이브 확인
6. [기록] 인시던트 기록 + 근본 원인 분석
```

### C. 롤백
```
1. [판단] 배포 후 5분 내 심각한 이슈 발견
2. [실행] Cloudflare Dashboard → 이전 배포 버전 선택 → Rollback
3. [확인] 롤백 후 서비스 정상 동작 확인
4. [분석] 원인 파악 후 재배포 계획 수립
```

## ✅ 배포 전 체크리스트 (복사용)
```markdown
## 배포 체크리스트 — [날짜]

### 코드
- [ ] TypeScript 타입 에러 0건
- [ ] console.log 제거
- [ ] 환경변수 하드코딩 없음

### 보안
- [ ] .env 파일 Git 미포함
- [ ] API 키 환경변수 처리
- [ ] CORS 설정 확인

### 성능
- [ ] 빌드 에러 없음
- [ ] Lighthouse 90+ 확인

### 기능
- [ ] 핵심 플로우 테스트 완료
- [ ] 모바일 화면 확인
- [ ] 에러 상태 처리 확인
```

## 📊 배포 후 모니터링 항목
| 시점 | 확인 항목 |
|------|----------|
| 5분 후 | 에러 로그, 서버 응답 코드 |
| 1시간 후 | 사용자 행동 (GA), 전환율 |
| 24시간 후 | 이탈률 변화, 성능 지표 |
| 1주일 후 | 전체 KPI 비교 (배포 전 vs 후) |
