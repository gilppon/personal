# 기술 스택 선택 가이드

> 넥스트하루 1인 기업에 최적화된 기술 스택 의사결정 가이드

## 🎯 선택 원칙
1. **최소 비용, 최대 효율**: 무료 티어가 넉넉한 서비스 우선
2. **학습 곡선 최소화**: React 생태계 통일
3. **원맨 운영 가능**: 관리 포인트 최소화 (서버리스/매니지드 우선)

## 🏗️ 프로젝트 유형별 추천 스택

### 웹앱 (SEO 필요)
```
Next.js (App Router) + Supabase + Cloudflare Pages
```
- SSR/SSG로 SEO 최적화
- Supabase Auth + DB + Storage 올인원
- Cloudflare 에지 배포로 글로벌 성능

### 웹앱 (SPA, SEO 불필요)
```
Vite + React + Supabase + Cloudflare Pages
```
- 빠른 HMR, 가벼운 번들
- 대시보드, 어드민, 내부 도구에 적합

### 모바일앱
```
Expo (React Native) + Supabase + EAS Build
```
- iOS/Android 동시 개발
- OTA 업데이트로 앱스토어 심사 우회
- EAS로 빌드/배포 자동화

### 정적 사이트 / 랜딩페이지
```
HTML + Vanilla CSS + Cloudflare Pages
```
- 프레임워크 없이 최대 성능
- 유지보수 포인트 제로

### API / 서버리스 함수
```
Cloudflare Workers + D1/KV
```
- 콜드 스타트 없음 (V8 isolate)
- 글로벌 에지에서 실행

## 💰 비용 비교 (월간, 2026 기준)

|--------|----------|----------|
| Cloudflare Pages | 무제한 대역폭 | - |
| Cloudflare Workers | 10만 req/일 | $5/월 |

## ⚠️ 피해야 할 선택
- AWS/GCP 직접 운영: 1인 기업에 과도한 관리 부담
- 유료 CMS: WordPress 호스팅 비용 + 보안 관리
- 자체 서버: 서버 관리 시간 = 제품 개발 시간 감소
