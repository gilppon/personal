# 성능 최적화 패턴

> 웹앱 성능을 극대화하는 실전 패턴 모음

## ⚡ 프론트엔드 최적화

### 이미지
- **포맷**: WebP (95% 브라우저 지원), AVIF (차세대)
- **로딩**: `loading="lazy"` 기본 적용 (뷰포트 밖 이미지)
- **크기**: 표시 크기의 2배 해상도까지만 (레티나 대응)
- **CDN**: Cloudflare 자동 이미지 최적화 활용

### 번들 최적화
```javascript
// ✅ Dynamic Import (코드 스플리팅)
const HeavyComponent = lazy(() => import('./HeavyComponent'));

// ✅ Tree Shaking 가능하게 named export 사용
export { specificFunction } from './utils';

// ❌ 전체 import 금지
// import _ from 'lodash'; → import { debounce } from 'lodash-es';
```

### 렌더링 최적화
```javascript
// ✅ 비싼 계산은 useMemo
const sorted = useMemo(() => items.sort(compareFn), [items]);

// ✅ 콜백 안정화는 useCallback
const handleClick = useCallback(() => { ... }, [dependency]);

// ✅ 리스트 렌더링은 고유 key 사용
{items.map(item => <Item key={item.id} />)}

// ❌ index를 key로 사용 금지 (순서 변경 시 버그)
```

### API 호출 최적화
- **디바운싱**: 검색 입력 300ms 디바운스
- **캐싱**: SWR/React Query로 자동 캐시 + 재검증
- **페이지네이션**: 무한 스크롤 또는 커서 기반 페이징
- **병렬 요청**: `Promise.all` 로 독립 API 동시 호출

## 🖥️ 서버 최적화

### Cloudflare Workers
- **KV 캐시**: 자주 읽는 데이터 KV에 저장 (읽기 초고속)
- **Cache API**: 응답 캐싱으로 Worker 실행 횟수 절약
- **Edge 로직**: 사용자와 가까운 에지에서 처리

### 데이터베이스
- **인덱스**: 자주 검색/정렬하는 컬럼에 인덱스 필수
- **N+1 문제**: JOIN 또는 관계 쿼리로 해결
- **커넥션 풀링**: Supabase Pooler 사용 (서버리스 환경)
