# 배포 체크리스트

> 서비스 배포 전 반드시 확인해야 할 항목 (Cloudflare 기반)

## 🔒 보안 체크
- [ ] 환경변수에 시크릿 저장 (하드코딩 없음)
- [ ] `.env` 파일이 `.gitignore`에 포함
- [ ] CORS 설정 명시적 도메인 지정 (`*` 금지)
- [ ] HTTPS 강제 적용
- [ ] API Rate Limiting 설정

## 🏗️ 빌드 체크
- [ ] `npm run build` 에러 없이 완료
- [ ] 빌드 결과물 크기 확인 (JS 번들 < 200KB 권장)
- [ ] 사용하지 않는 의존성 제거
- [ ] TypeScript 타입 에러 0건

## 📊 성능 체크
- [ ] Lighthouse 점수 90+ (Performance, Accessibility, SEO)
- [ ] 이미지 WebP/AVIF 변환 완료
- [ ] 폰트 subset + display:swap 적용
- [ ] Critical CSS 인라인 처리 (또는 자동 추출)
- [ ] 불필요한 console.log 제거

## 🔍 SEO 체크 (웹앱인 경우)
- [ ] `<title>`, `<meta description>` 모든 페이지에 적용
- [ ] Open Graph 태그 (og:title, og:image, og:description)
- [ ] sitemap.xml 생성
- [ ] robots.txt 설정
- [ ] 구조화 데이터 (Schema.org) 적용

## 🚀 배포 절차
1. `git push` → CI/CD 자동 빌드
2. Preview URL에서 기능 테스트
3. 핵심 플로우 수동 테스트 (가입, 로그인, 결제 등)
4. Production 배포
5. 배포 후 5분 모니터링 (에러 로그 확인)
6. `_shared/decisions.md`에 배포 기록

## ⚠️ 롤백 절차
1. Cloudflare Dashboard → Deployments → 이전 버전 선택
2. "Rollback" 클릭
3. 원인 분석 후 핫픽스 적용
