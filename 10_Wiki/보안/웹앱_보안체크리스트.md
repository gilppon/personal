# 웹앱 보안 체크리스트

> 배포 전 반드시 확인해야 할 보안 항목

## 🔒 인증/인가 (Authentication & Authorization)

### 필수 항목
- [ ] 비밀번호 해싱 (bcrypt, argon2 — 절대 평문 저장 금지)
- [ ] JWT 만료 시간 설정 (Access: 15분, Refresh: 7일)
- [ ] 세션/토큰 탈취 대비 (HttpOnly, Secure, SameSite 쿠키)
- [ ] 권한 검사: API마다 사용자 권한 확인 (프론트 체크만으로 불충분)
- [ ] Rate Limiting: 로그인 시도 5회 실패 → 30분 잠금

## 🛡️ 데이터 보호

### SQL Injection 방지
```javascript
// ❌ 절대 금지
const query = `SELECT * FROM users WHERE id = '${userId}'`;

// ✅ 파라미터 바인딩
const { data } = await supabase.from('users').select('*').eq('id', userId);
```

### XSS (Cross-Site Scripting) 방지
- 사용자 입력 출력 시 HTML 이스케이프 처리
- React는 기본적으로 이스케이프하지만 `dangerouslySetInnerHTML` 사용 금지
- Content Security Policy (CSP) 헤더 설정

### CSRF (Cross-Site Request Forgery) 방지
- SameSite 쿠키 속성 설정
- Anti-CSRF 토큰 사용 (폼 제출 시)

## 🔑 시크릿 관리

### 절대 금지
- ❌ 코드에 API 키 하드코딩
- ❌ `.env` 파일 Git에 커밋
- ❌ 클라이언트 코드에 서버 시크릿 노출
- ❌ 에러 메시지에 내부 정보 포함

### 올바른 관리
- ✅ 환경변수로 관리 (`.env.local`)
- ✅ `.gitignore`에 `.env*` 패턴 추가
- ✅ 프론트엔드용 키는 `NEXT_PUBLIC_` 접두사로 구분
- ✅ 정기적 키 로테이션 (최소 90일)

## 🌐 네트워크 보안
- [ ] HTTPS 강제 (Cloudflare 기본 적용)
- [ ] CORS: 허용 도메인 명시적 지정
- [ ] API 응답에 불필요한 헤더 제거 (X-Powered-By 등)
- [ ] 파일 업로드: 확장자/크기/MIME 타입 검증

## 📋 정기 보안 점검
- **월간**: 의존성 취약점 스캔 (`npm audit`)
- **분기별**: 접근 권한 리뷰, 미사용 API 키 삭제
- **연간**: 보안 정책 전체 리뷰
