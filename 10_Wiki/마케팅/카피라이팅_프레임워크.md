# 카피라이팅 프레임워크

> 전환율을 높이는 카피라이팅 공식 모음

## 📝 3대 프레임워크

### AIDA (주의 → 흥미 → 욕구 → 행동)
```
[주의] 아직도 매일 5시간씩 반복 작업을 하세요?
[흥미] 1인 개발자 500명이 이 도구로 시간을 절반으로 줄였습니다.
[욕구] 남는 시간에 새 프로젝트를 시작하거나, 그냥 쉬세요.
[행동] 지금 무료로 시작하기 →
```

### PAS (문제 → 자극 → 해결)
```
[문제] 코드 배포할 때마다 불안하지 않으세요?
[자극] 한 번의 실수로 서비스 전체가 다운됩니다. 새벽 3시에 전화가 올 수도 있죠.
[해결] 자동 롤백 시스템으로 밤에도 편히 주무세요.
```

### BAB (이전 → 이후 → 다리)
```
[이전] 매주 보고서 만드는 데 반나절을 쓰던 시절.
[이후] 버튼 하나로 자동 생성되는 보고서.
[다리] 넥스트하루 대시보드가 그 다리입니다.
```

## 🎯 용도별 카피 가이드

### 랜딩페이지 히어로
- **제목**: 혜택 중심 (기능 X, 결과 O)
- **부제목**: 구체적 수치 또는 대상 명시
- **CTA 버튼**: 동사로 시작 ("시작하기", "무료로 체험")
- ❌ "제출", "클릭" 같은 무미건조한 CTA 금지

### SNS 캡션
- 첫 줄에 Hook (스크롤 멈추기)
- 3줄 이내로 핵심 전달
- 마지막에 CTA ("저장 📌", "댓글 💬")
- 이모지는 강조용으로만 (과다 사용 금지)

### 이메일
- **제목**: 개인적 톤 + 호기심 유발 (클릭률 결정)
- **본문**: 첫 문장에서 "왜 이 메일을 열어야 하는지"
- **CTA**: 버튼 1개만 (선택지 많으면 전환율 하락)

## ✍️ 넥스트하루 톤앤매너 규칙
1. **직설적**: 돌려 말하지 않는다
2. **실용적**: 미사여구 없이 핵심만
3. **인간적**: "저희"보다 "우리", 경험 녹이기
4. **자신감**: 겸손한 척 금지, 확신 있게
5. **구어체**: 문어체보다 말하듯 자연스럽게
