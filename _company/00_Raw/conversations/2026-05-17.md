## 1. 프로젝트 개요 (Project Overview)
* **목적:** 기업이 보유한 원시 문서(PDF, Excel, TXT 등)를 입력하면, 하이드로/로컬 환경에서 데이터 가공부터 소형 언어 모델(SLM) 학습 및 배포까지 원스톱으로 수행하는 "자동화된 맞춤형 SLM 빌더 프로그램" 개발.
* **타겟 시장 환경 (2026년 기준):** 고비용 LLM(거대 언어 모델) 대비 비용 효율성(최대 90% 절감)이 높고 사내 보안(온프레미스) 유지가 가능한 3B~8B 규모의 도메인 특화 SLM 수요 폭발.

## 2. 핵심 파이프라인 아키텍처 (Core Pipeline Architecture)

### [Phase 1: 합성 데이터 생성 모듈 (Synthetic Data Generation)]
* **기능:** 가공되지 않은 사내 문서를 AI 학습용 Q&A 데이터셋으로 자동 변환.
* **프로세스:**
  1. **문서 파싱:** PDF, XLSX, DOCX, TXT 파일의 텍스트 자동 추출.
  2. **텍스트 청킹(Chunking):** 문맥을 유지하며 1,000자 내외의 의미 단위로 단락 분할.
  3. **Q&A 프롬프팅:** 고성능 LLM(프론티어 API 또는 로컬 헤비 모델)을 활용해 해당 단락 기반의 예상 질문 및 모범 답변 쌍(ChatML 규격) 자동 생성.
  4. **데이터셋 내보내기:** 파인튜닝용 규격 파일(`dataset.json` 또는 `dataset.jsonl`)로 저장.

### [Phase 2: 고속 파인튜닝 모듈 (Automated Fine-Tuning)]
* **기능:** 생성된 데이터셋을 기반으로 오픈소스 SLM을 기업 맞춤형으로 미세조정 학습.
* **기술 스택:** `Unsloth`, Hugging Face `PEFT` (LoRA / QLoRA 라이브러리 활용).
* **대상 기반 모델 (Base Models):** Llama 3, Gemma 2, Phi 등 최신 오픈소스 SLM (3B ~ 8B 파라미터).
* **특징:** 메모리(VRAM) 소모 최소화, 학습 속도 극대화, 로컬 GPU 환경 최적화.

### [Phase 3: 모델 최적화 및 양자화 (Quantization & Export)]
* **기능:** 학습 완료된 모델을 일반 사내 PC 및 서버에서 가볍게 구동할 수 있도록 압축.
* **포맷:** `GGUF` 또는 `Safetensors` 포맷으로 자동 변환 및 양자화(INT4/INT8) 적용.
* **호환성:** `Ollama`, `LM Studio` 등 로컬 런타임 엔진과 즉시 연동 가능하도록 설계.

### [Phase 4: 로컬 서빙 및 UI 모듈 (Local Serving & Interface)]
* **기능:** 사내 직원들이 학습된 SLM과 웹 브라우저를 통해 대화할 수 있는 인터페이스 제공.
* **기술 스택:** `Gradio` 또는 `Streamlit` 기반의 경량 웹 UI 구축.

import os
import json
import pandas as pd
from pypdf import PdfReader
from typing import List, Dict, Tuple
import time

# --- Phase 2: Fine-Tuning & LLM Libraries ---
# 실제 환경에서는 아래 라이브러리들이 설치되어 있어야 합니다.
try:
    from unsloth import FastLanguageModel, FastLanguageModelForCausalLM
    from transformers import AutoTokenizer, BitsAndBytesConfig, TrainingArguments
    from trl import SFTTrainer
except ImportError as e:
    print(f"🚨 필수 라이브러리 누락: {e}. 설치를 확인해주세요.")

# --- Phase 1: Data Generation Libraries ---
def extract_text_from_pdf(pdf_path: str) -> str:
    """PDF 파일에서 텍스트를 추출합니다."""
    print(f"📄 PDF 파일 읽기 시작: {pdf_path}")
    text = ""
    try:
        reader = PdfReader(pdf_path)
        for page in reader.pages:
            text += page.extract_text() + "\n\n"
        print("✅ PDF 텍스트 추출 완료.")
        return text
    except Exception as e:
        print(f"❌ PDF 파일 처리 중 오류 발생 ({pdf_path}): {e}")
        return ""

def chunk_text(text: str, chunk_size: int = 1000, overlap: int = 200) -> List[str]:
    """텍스트를 의미 단위로 청킹합니다."""
    print(f"✂️ 텍스트 청킹 시작 (Chunk Size: {chunk_size}, Overlap: {overlap})")
    chunks = []
    start = 0
    while start < len(text):
        end = min(start + chunk_size, len(text))
        chunk = text[start:end]
        chunks.append(chunk.strip())
        # 다음 청킹 시작 위치 계산 (겹침 적용)
        if end == len(text):
            break
        start += (chunk_size - overlap)
    print(f"✅ 총 {len(chunks)}개의 청크 생성 완료.")
    return chunks

def generate_qa_pairs(chunks: List[str], model_name: str = "Llama-3-8B") -> List[Dict]:
    """청크를 기반으로 Q&A 쌍을 생성합니다. (LLM 호출 시뮬레이션)"""
    print("🧠 Q&A 쌍 생성 시작...")
    qa_data = []
    for i, chunk in enumerate(chunks):
        # 실제 환경에서는 이 부분을 LLM API 호출로 대체해야 합니다.
        # 여기서는 프로토타입을 위해 임시 프롬프트와 더미 응답을 사용합니다.
        prompt = f"다음 텍스트를 기반으로, 핵심 정보를 요약하고 3개의 질문과 그에 대한 답변(ChatML 형식)을 생성하세요. 텍스트: \"{chunk[:500]}...\""

        # --- LLM API 호출 시뮬레이션 시작 ---
        if "Llama-3-8B" in model_name:
            # Llama 3/Gemma를 가정하고 더미 응답 생성
            mock_qa = [
                {"question": f"Chunk {i}의 핵심 내용은 무엇인가요?", "answer": f"이 청크는 문서의 특정 단락을 포함하며, 주요 정보는 {chunk[:100]}에 있습니다."},
                {"question": f"이 내용에서 가장 중요한 사실은 무엇인가요?", "answer": "핵심 정보는 주어진 텍스트 내에 명시되어 있습니다."},
                {"question": f"추가로 알면 좋을 점은 무엇인가요?", "answer": "전체 문맥을 이해하기 위해서는 다음 청크와 연결해야 합니다."}
            ]
        else: # Gemma 또는 기타 모델 가정
             mock_qa = [
                {"question": f"Chunk {i}의 핵심 내용은 무엇인가요?", "answer": f"이 청크는 문서의 특정 단락을 포함하며, 주요 정보는 {chunk[:100]}에 있습니다."},
                {"question": f"이 내용에서 가장 중요한 사실은 무엇인가요?", "answer": "핵심 정보는 주어진 텍스트 내에 명시되어 있습니다."},
                {"question": f"추가로 알면 좋을 점은 무엇인가요?", "answer": "전체 문맥을 이해하기 위해서는 다음 청크와 연결해야 합니다."}
            ]
        # --- LLM API 호출 시뮬레이션 종료 ---

        qa_data.extend(mock_qa)
        time.sleep(0.1) # API 지연 시뮬레이션

    print(f"✅ 총 {len(qa_data)}개의 Q&A 쌍 생성 완료.")
    return qa_data

def save_dataset(qa_data: List[Dict], output_path: str):
    """생성된 Q&A 데이터셋을 JSON 파일로 저장합니다."""
    print(f"💾 데이터셋 저장 시작: {output_path}")
    with open(output_path, 'w', encoding='utf-8') as f:
        json.dump(qa_data, f, ensure_ascii=False, indent=4)
    print("✅ 데이터셋 저장 완료.")

# --- Phase 2: Fine-Tuning Module ---
def fine_tune_model(train_file: str, model_name: str = "Llama-3-8B"):
    """Unsloth와 PEFT를 사용하여 LoRA 파인튜닝을 수행합니다."""
    print("\n🚀 Phase 2: Unsloth/PEFT 기반 LoRA 파인튜닝 시작")

    # 1. 모델 및 토크나이저 로드 (Unsloth 사용)
    print(f"⚙️ 모델 로딩 중: {model_name}")
    bnb_config = BitsAndBytesConfig(
        load_in_4bit=True,
        bnb_4bit_quant_type="nf4",
        bnb_4bit_compute_dtype=torch.bfloat16
    )

    # Unsloth를 사용하여 모델 로드 및 튜닝 준비
    model, tokenizer = FastLanguageModel.from_pretrained(
        model_name,
        bnb_config=bnb_config,
        token=os.getenv("HF_TOKEN"), # 토큰 환경변수 사용 권장
        device_map="auto"
    )

    # 2. 데이터셋 로드 및 토크나이저 설정
    dataset = pd.read_json(train_file)
    print(f"📚 데이터셋 로드 완료. 총 {len(dataset)}개의 샘플.")

    # 3. LoRA 설정 (PEFT)
    # Unsloth는 FastLanguageModel을 사용하므로 PEFT 설정은 내부적으로 통합되거나 간소화됨.
    # 여기서는 SFTTrainer를 사용하여 학습을 진행합니다.
    output_dir = "./results"
    os.makedirs(output_dir, exist_ok=True)

    training_args = TrainingArguments(
        per_device_train_batch_size=4,
        gradient_accumulation_steps=4,
        warmup_steps=10,
        max_steps=50, # 프로토타입을 위해 스텝 수를 제한합니다. 실제 학습은 더 길어야 합니다.
        learning_rate=2e-4,
        logging_steps=10,
        output_dir=output_dir,
        fp16=True,
        save_strategy="steps",
        save_steps=50,
    )

    # 4. SFTTrainer 설정 및 학습 실행
    trainer = SFTTrainer(
        model=model,
        tokenizer=tokenizer,
        train_dataset=dataset,
        dataset_text_field="text", # Q&A 쌍의 텍스트를 사용할 필드 지정
        max_seq_length=1024,
        args=training_args,
        packing=False,
    )

    print("▶️ LoRA 학습 시작...")
    start_time = time.time()
    trainer.train()
    end_time = time.time()

    print(f"🎉 ✨ 파인튜닝 완료! 총 소요 시간: {end_time - start_time:.2f}초")

    # 5. 모델 저장 (LoRA 어댑터)
    output_model_path = os.path.join(output_dir, "final_adapter")
    trainer.save_model(output_model_path)
    print(f"💾 최종 LoRA 어댑터가 다음 경로에 저장되었습니다: {output_model_path}")


# --- Main Execution Flow ---

# 실제 환경에서는 아래 라이브러리들이 설치되어 있어야 합니다.

def extract_text_from_pdf(pdf_path: str) -> str:
    """PDF 파일에서 텍스트를 추출합니다."""
    print(f"📄 PDF 파일 읽기 시작: {pdf_path}")
    text = ""
    try:
        reader = PdfReader(pdf_path)
        for page in reader.pages:
            text += page.extract_text() + "\n\n"
        print("✅ PDF 텍스트 추출 완료.")
        return text
    except Exception as e:
        print(f"❌ PDF 파일 처리 중 오류 발생 ({pdf_path}): {e}")
        return ""

def chunk_text(text: str, chunk_size: int = 1000, overlap: int = 200) -> List[str]:
    """텍스트를 의미 단위로 청킹합니다."""
    print(f"✂️ 텍스트 청킹 시작 (Chunk Size: {chunk_size}, Overlap: {overlap})")
    chunks = []
    start = 0
    while start < len(text):
        end = min(start + chunk_size, len(text))
        chunk = text[start:end]
        chunks.append(chunk.strip())
        # 다음 청킹 시작 위치 계산 (겹침 적용)
        if end == len(text):
            break
        start += (chunk_size - overlap)
    print(f"✅ 총 {len(chunks)}개의 청크 생성 완료.")
    return chunks

def generate_qa_pairs(chunks: List[str], model_name: str = "Llama-3-8B") -> List[Dict]:
    """청크를 기반으로 Q&A 쌍을 생성합니다. (LLM 호출 시뮬레이션)"""
    print("🧠 Q&A 쌍 생성 시작...")
    qa_data = []
    for i, chunk in enumerate(chunks):
        # 실제 환경에서는 이 부분을 LLM API 호출로 대체해야 합니다.
        # 여기서는 프로토타입을 위해 임시 프롬프트와 더미 응답을 사용합니다.
        prompt = f"다음 텍스트를 기반으로, 핵심 정보를 요약하고 3개의 질문과 그에 대한 답변(ChatML 형식)을 생성하세요. 텍스트: \"{chunk[:500]}...\""

        # --- LLM API 호출 시뮬레이션 시작 ---

                {"question": f"이 내용에서 가장 중요한 사실은 무엇인가요?", "answer": "핵심 정보는 주어진 텍스트 내에 명시되어 있습니다."},
                {"question": f"추가로 알면 좋을 점은 무엇인가요?", "answer": "전체 문맥을 이해하기 위해서는 다음 청크와 연결해야 합니다."}
            ]
        else: # Gemma 또는 기타 모델 가정
             mock_qa = [
                {"question": f"Chunk {i}의 핵심 내용은 무엇인가요?", "answer": f"이 청크는 문서의 특정 단락을 포함하며, 주요 정보는 {chunk[:100]}에 있습니다."},
                {"question": f"이 내용에서 가장 중요한 사실은 무엇인가요?", "answer": "핵심 정보는 주어진 텍스트 내에 명시되어 있습니다."},
                {"question": f"추가로 알면 좋을 점은 무엇인가요?", "answer": "전체 문맥을 이해하기 위해서는 다음 청크와 연결해야 합니다."}
            ]
        # --- LLM API 호출 시뮬레이션 종료 ---

        qa_data.extend(mock_qa)
        time.sleep(0.1) # API 지연 시뮬레이션

    print(f"✅ 총 {len(qa_data)}개의 Q&A 쌍 생성 완료.")
    return qa_data

def save_dataset(qa_data: List[Dict], output_path: str):
    """생성된 Q&A 데이터셋을 JSON 파일로 저장합니다."""
    print(f"💾 데이터셋 저장 시작: {output_path}")
    with open(output_path, 'w', encoding='utf-8') as f:
        json.dump(qa_data, f, ensure_ascii=False, indent=4)
    print("✅ 데이터셋 저장 완료.")

# --- Phase 2: Fine-Tuning Module ---
def fine_tune_model(train_file: str, model_name: str = "Llama-3-8B"):
    """Unsloth와 PEFT를 사용하여 LoRA 파인튜닝을 수행합니다."""
    print("\n🚀 Phase 2: Unsloth/PEFT 기반 LoRA 파인튜닝 시작")

    # 1. 모델 및 토크나이저 로드 (Unsloth 사용)
    print(f"⚙️ 모델 로딩 중: {model_name}")
    bnb_config = BitsAndBytesConfig(
        load_in_4bit=True,
        bnb_4bit_quant_type="nf4",
        bnb_4bit_compute_dtype=torch.bfloat16
    )

    # Unsloth를 사용하여 모델 로드 및 튜닝 준비
    model, tokenizer = FastLanguageModel.from_pretrained(
        model_name,
        bnb_config=bnb_config,
        token=os.getenv("HF_TOKEN"), # 토큰 환경변수 사용 권장
        device_map="auto"
    )

    # 2. 데이터셋 로드 및 토크나이저 설정
    dataset = pd.read_json(train_file)
    print(f"📚 데이터셋 로드 완료. 총 {len(dataset)}개의 샘플.")

    # 3. LoRA 설정 (PEFT)
    # Unsloth는 FastLanguageModel을 사용하므로 PEFT 설정은 내부적으로 통합되거나 간소화됨.
    # 여기서는 SFTTrainer를 사용하여 학습을 진행합니다.
    output_dir = "./results"
    os.makedirs(output_dir, exist_ok=True)

    training_args = TrainingArguments(
        per_device_train_batch_size=4,
        gradient_accumulation_steps=4,
        warmup_steps=10,
        max_steps=50, # 프로토타입을 위해 스텝 수를 제한합니다. 실제 학습은 더 길어야 합니다.
        learning_rate=2e-4,
        logging_steps=10,
        output_dir=output_dir,
        fp16=True,
        save_strategy="steps",
        save_steps=50,
    )

    # 4. SFTTrainer 설정 및 학습 실행
    trainer = SFTTrainer(
        model=model,
        tokenizer=tokenizer,
        train_dataset=dataset,
        dataset_text_field="text", # Q&A 쌍의 텍스트를 사용할 필드 지정
        max_seq_length=1024,
        args=training_args,
        packing=False,
    )

    print("▶️ LoRA 학습 시작...")
    start_time = time.time()
    trainer.train()
    end_time = time.time()

    print(f"🎉 ✨ 파인튜닝 완료! 총 소요 시간: {end_time - start_time:.2f}초")

    # 5. 모델 저장 (LoRA 어댑터)
    output_model_path = os.path.join(output_dir, "final_adapter")
    trainer.save_model(output_model_path)
    print(f"💾 최종 LoRA 어댑터가 다음 경로에 저장되었습니다: {output_model_path}")
