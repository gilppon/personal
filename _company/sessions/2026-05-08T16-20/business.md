# 📋 프로토타입 검증 계획서 (QA & UX Audit)
### 목표: A/B 테스트 기반 CTA의 완벽한 작동 및 사용자 심리적 이탈 방지 확인

## I. 사용자 경험(UX) 시나리오 체크리스트 (Persona-Based Flow)
**목표:** 사용자가 'Pain'을 느끼고 → 'C (Hybrid)' 솔루션에 설득되어 → 버튼을 클릭하는 과정의 마찰 요소를 점검합니다.

| 순서 | 테스트 영역 | 시나리오 액션 (사용자 관점) | 기대 결과 (Pass Criteria) | 리스크 유형 |
| :--- | :--- | :--- | :--- | :--- |
| **1** | **Pain 인식 유도 (Top)** | 페이지 진입 후, "바이브코딩 오류/홍보 지식 부족"이라는 Pain 포인트를 명확하게 인지하는가? | 헤드라인과 서브 텍스트를 읽는 순간, '이건 나한테 하는 이야기다'라는 즉각적인 공감을 느껴야 함. (감정적 Hook 작동) | **인지 부조화** |
| **2** | **솔루션 제시 (Middle)** | CTA 직전에 제공되는 가치 제안(Value Proposition)을 읽었을 때, 문제의 해결책이 구체적으로 와닿는가? | '어떻게?'라는 질문에 대해 `최소 비용 고부가 가치 웹앱`이라는 명확하고 실행 가능한 답을 얻어야 함. (추상적 설명 금지) | **불명료성** |
| **3** | **CTA 압박감 조성 (Button)** | CTA 버튼 영역에 도달했을 때, 감정적으로 '지금 아니면 안 될 것 같은' 긴장감을 느끼는가? | Z-Axis 깊이와 HSL 색상 대비를 통해 시각적 우선순위(Visual Hierarchy) 1위를 차지해야 함. **망설임을 제로화**할 정도의 압박감 유도. | **관심 분산/지루함** |
| **4** | **CTA 메시지 확인 (Copy)** | 최종 CTA 문구(`C (Hybrid)`: 예시 "지금, 1시간 만에 시작하세요")를 읽었을 때, 망설임 없이 다음 행동이 결정되는가? | 시간적 손실(Time-to-Loss)과 즉각적인 이득(Immediate Gain)이 명확하게 연결되어야 함. (예: '잃기 전에 잡는다') | **행동 지연** |
| **5** | **후속 조치 시뮬레이션 (Click)** | CTA 버튼을 클릭했을 때, 다음 페이지로 이동하는 과정이 매끄럽고 기대했던 행동(폼 제출/가입)으로 이어지는가? | 로딩 시간은 0.3초 이내여야 하며, 가입 흐름의 첫 단계에서 추가적인 정보 입력 장벽이 발생하지 않아야 함. (Frictionless UX) | **사용자 이탈** |

## II. 기능 및 기술 QA 계획 (Technical Validation)
**목표:** 디자인 명세가 코드로 완벽하게 구현되었는지, 특히 데이터 추적과 스타일링이 깨지지 않았는지를 검증합니다.

### 🛠️ A. 백엔드/데이터 무결성 테스트 (Developer & QA)
| 체크 항목 | 테스트 시나리오 | 기대 결과 (Pass Criteria) | 중요도 |
| :--- | :--- | :--- | :--- |
| **A/B 테스트 추적** | CTA 클릭을 3가지 버전(A, B, C)으로 순차적으로 수행. | `user_events` 테이블에 `event_id=CTA_Test_[A|B|C]`가 정확히 기록되며, 각 이벤트는 유일한 세션 ID와 연결되어야 함. | ⭐⭐⭐ (최고) |
| **RLS 정책 준수** | 비인가 사용자(로그아웃 상태 등)로 접근하여 CTA 클릭을 시도. | 데이터베이스에 아무런 쓰기(Write)가 발생하지 않거나, 오직 `user_id`를 NULL 처리한 *익명* 이벤트만 기록되어야 함. (보안성 확보) | ⭐⭐⭐ (최고) |
| **데이터 적시성** | 테스트 직후 개발 콘솔에서 데이터를 조회. | 데이터 삽입(Insert) 시점의 타임스탬프가 지연 없이 실시간으로 반영되는지 확인해야 함. | ⭐⭐ |

### 🎨 B. 프론트엔드/디자인 구현 검증 (Designer & QA)
| 체크 항목 | 테스트 시나리오 | 기대 결과 (Pass Criteria) | 중요도 |
| :--- | :--- | :--- | :--- |
| **Z-Axis 깊이** | CTA 컴포넌트를 다양한 배경색(어두운색/밝은색) 위에 배치하여 확인. | 버튼이 물리적으로 떠오르는 듯한 입체감(`translateZ`)과 그림자 효과(`shadow-depth`)가 모든 환경에서 일관되게 유지되어야 함. | ⭐⭐⭐ (최고) |
| **HSL 컬러 변화** | `:hover` 및 `:active` 상태에서 색상(Blue 계열)이 정의된 HSL 변수만큼 부드럽게 변화하는지 확인. | 단순한 색상 변경이 아닌, 빛의 반사각이 변하는 듯한 '유기적인' 전환 애니메이션(Micro-interaction)을 보여야 함. | ⭐⭐ |
| **반응형 디자인** | 모바일(375px), 태블릿(800px), 데스크탑(1440px) 세 가지 해상도에서 확인. | 버튼의 폰트 크기, 패딩, 전체 너비가 각 기기에 최적화되며 레이아웃이 깨지지 않아야 함. (특히 CTA 주변 여백 확보 중요) | ⭐⭐⭐ (최고) |

**[QA 결론 및 권고]**

1.  **Critical Path:** A/B 테스트의 **'데이터 추적 무결성(A)'**과 UX의 **'긴장감 조성(UX 3, 4)'**이 가장 치명적인 실패 지점입니다. 이 두 가지가 완벽히 통과하지 못하면 마케팅 자원 투입 자체가 의미 없습니다.
2.  **자동화 방안:** 버튼 클릭 후 발생하는 '사용자 플로우'는 **Zapier 또는 Webhook을 활용한 자동 가입 흐름 검증 툴**로 대체 테스트하는 것을 고려해야 합니다. (수동 QA 시간 절약 및 반복 작업 제거)
