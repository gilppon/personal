## 📐 CTA 컴포넌트 시각 디자인 명세 (CSS 변수 및 레이아웃 좌표)

본 명세는 `components/HeroSection.tsx`에서 사용된 색상과 공간감을 재사용 가능한 CSS 변수로 정의하여, 개발팀이 즉시 적용할 수 있도록 합니다. 모든 여백은 8px 그리드 시스템을 준수합니다.

### 1. 🎨 HSL 컬러 시스템 (Semantic Color Palette)

| 변수명 | 역할 (UX 목적) | HSL 값 (Pain-First/Solution-First) | CSS 속성 적용 예시 |
| :--- | :--- | :--- | :--- |
| `--color-pain` | 사용자의 손실(Pain Point) 강조 (배경/상단 영역) | `hsl(210, 85%, 7.5%)` | `background-color: var(--color-pain);` |
| `--color-solution` | 해결책 제시 및 행동 유도 (CTA 버튼) | `hsl(240, 90%, 65%)` | `background-color: var(--color-solution);` |
| `--color-text` | 핵심 정보 전달 (가독성 확보) | `hsl(210, 30%, 15%)` | `color: var(--color-text);` |
| `--shadow-depth` | Z-Axis 공간감 및 깊이 표현을 위한 그림자 | `rgba(0, 0, 0, 0.2)` | `box-shadow: 0 10px 30px var(--shadow-depth);` |

### 2. 📏 레이아웃 좌표 시스템 (8px 그리드 준수)

모든 패딩과 마진은 8의 배수를 사용합니다.

| 요소 | 속성 | 값 (px) | 설명 및 공간감 적용 방식 |
| :--- | :--- | :--- | :--- |
| **전체 컨테이너** | `padding` (Y축) | `80px` | 상하의 시각적 여백을 확보하여 깊이감을 부여합니다. |
| | `padding` (X축) | `40px` | 좌우 여백을 균형 있게 유지합니다. |
| **헤드라인 (`h1`)** | `font-size` | `3.5rem` | 시각적 무게 중심을 잡습니다. |
| | `margin-bottom` | `80px` | 다음 섹션과의 공간적 분리를 명확히 합니다. |
| **CTA 버튼 (컨테이너)** | `padding` (Y축) | `24px` | CTA 영역에 충분한 터치 영역을 제공합니다. |
| | `margin-top` | `32px` | 상단 텍스트와의 간격을 확보하여 위계를 설정합니다. |
| **CTA 버튼 (`a` 태그)** | `border-radius` | `4px` | 미세한 부드러움(Tactile Texture)을 부여합니다. |

### 3. ⚙️ Z-Axis 및 스타일링 상세 지침 (개발팀 전달용)

**목표:** 사용자가 '손실'에서 '해결책'으로 이동하는 심리적 흐름에 따라 레이어의 깊이를 표현합니다.

1.  **배경 처리 (`--color-pain` 적용 영역):**
    *   최상위 배경은 `--color-pain`을 사용하되, 실제 콘텐츠 영역 뒤에 미세한 블러 효과를 위한 `backdrop-filter: blur(4px)` 또는 다층 그림자(`box-shadow`)를 사용하여 요소가 화면 위에 떠 있다는 느낌을 구현합니다.
2.  **CTA 강조 (`--color-solution` 적용):**
    *   CTA 버튼은 가장 높은 시각적 우선순위를 가지며, 주변의 '고통' 영역과 명확히 분리되어야 합니다. `box-shadow`는 `--shadow-depth`를 활용하여 물리적인 깊이감을 부여합니다.
3.  **텍스트 (`--color-text` 적용):**
    *   모든 텍스트는 `--color-text`를 사용하여, 배경의 복잡성 속에서도 정보 전달의 명확성을 극대화해야 합니다 (가독성 최우선).
