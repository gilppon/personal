# 📋 작업 브리프

**원 명령:** # SYSTEM CONTEXT & PROJECT SPECIFICATION: AUTOMATED LOCAL SLM BUILDER

## 1. 프로젝트 개요 (Project Overview)
* **목적:** 기업이 보유한 원시 문서(PDF, Excel, TXT 등)를 입력하면, 하이드로/로컬 환경에서 데이터 가공부터 소형 언어 모델(SLM) 학습 및 배포까지 원스톱으로 수행하는 "자동화된 맞춤형 SLM 빌더 프로그램" 개발.
* **타겟 시장 환경 (2026년 기준):** 고비용 LLM(거대 언어 모델) 대비 비용 효율성(최대 90% 절감)이 높고 사내 보안(온프레미스) 유지가 가능한 3B~8B 규모의 도메인 특화 SLM 수요 폭발.

---

## 2. 핵심 파이프라인 아키텍처 (Core Pipeline Architecture)

### [Phase 1: 합성 데이터 생성 모듈 (Synthetic Data Generation)]
* **기능:** 가공되지 않은 사내 문서를 AI 학습용 Q&A 데이터셋으로 자동 변환.
* **프로세스:**
  1. **문서 파싱:** PDF, XLSX, DOCX, TXT 파일의 텍스트 자동 추출.
  2. **텍스트 청킹(Chunking):** 문맥을 유지하며 1,000자 내외의 의미 단위로 단락 분할.
  3. **Q&A 프롬프팅:** 고성능 LLM(프론티어 API 또는 로컬 헤비 모델)을 활용해 해당 단락 기반의 예상 질문 및 모범 답변 쌍(ChatML 규격) 자동 생성.
  4. **데이터셋 내보내기:** 파인튜닝용 규격 파일(`dataset.json` 또는 `dataset.jsonl`)로 저장.

### [Phase 2: 고속 파인튜닝 모듈 (Automated Fine-Tuning)]
* **기능:** 생성된 데이터셋을 기반으로 오픈소스 SLM을 기업 맞춤형으로 미세조정 학습.
* **기술 스택:** `Unsloth`, Hugging Face `PEFT` (LoRA / QLoRA 라이브러리 활용).
* **대상 기반 모델 (Base Models):** Llama 3, Gemma 2, Phi 등 최신 오픈소스 SLM (3B ~ 8B 파라미터).
* **특징:** 메모리(VRAM) 소모 최소화, 학습 속도 극대화, 로컬 GPU 환경 최적화.

### [Phase 3: 모델 최적화 및 양자화 (Quantization & Export)]
* **기능:** 학습 완료된 모델을 일반 사내 PC 및 서버에서 가볍게 구동할 수 있도록 압축.
* **포맷:** `GGUF` 또는 `Safetensors` 포맷으로 자동 변환 및 양자화(INT4/INT8) 적용.
* **호환성:** `Ollama`, `LM Studio` 등 로컬 런타임 엔진과 즉시 연동 가능하도록 설계.

### [Phase 4: 로컬 서빙 및 UI 모듈 (Local Serving & Interface)]
* **기능:** 사내 직원들이 학습된 SLM과 웹 브라우저를 통해 대화할 수 있는 인터페이스 제공.
* **기술 스택:** `Gradio` 또는 `Streamlit` 기반의 경량 웹 UI 구축.

## 3. 구축 및 배포 옵션 (Deployment Options)

### 옵션 A: 하이브리드 방식 (최고 효율/고성능)
* **데이터 생성:** 외부 고성능 API (Gemini API, OpenAI API 등)를 활용해 최고 품질의 Q&A 데이터셋 생성.
* **학습 및 구동:** 학습용 데이터셋을 로컬로 다운로드한 후, 사내 GPU 서버에서 Unsloth로 학습 및 로컬 배포.
* **장점:** 데이터 품질 우수, 학습 및 추론 비용 제로, 최종 모델의 완벽한 사내 보안 보장.

### 옵션 B: 100% 폐쇄망 방식 (완벽한 보안)
* **데이터 생성:** 사내 서버에 업로드한 다소 무거운 로컬 모델(예: Llama 3 70B)을 통해 Q&A 생성.
* **학습 및 구동:** 전 과정(데이터 생성 -> 학습 -> 배포)이 외부 인터넷 연결 없이 100% 사내 인프라 내에서만 진행.
* **장점:** 데이터 외부 유출 가능성 0% (금융, 의료, 공공기관 적합).

## 4. 인프라 및 기술적 요구사항 (Technical Requirements)

* **하드웨어 (로컬 학습 기준):**
  * 최소 사양: NVIDIA RTX 3090 / 4090 (VRAM 24GB) 1대
  * 권장 사양: NVIDIA A100 / H100 또는 클라우드 GPU 인스턴스 (학습 기간 대여)
* **필수 라이브러리:** `unsloth`, `transformers`, `trl`, `peft`, `accelerate`, `pypdf`, `pandas`, `gradio`
* **데이터 기준:** 최소 수백 ~ 수천 개 이상의 고품질 '질문-답변' 쌍 또는 일관성 있는 사내 텍스트 데이터 확보 필요.
